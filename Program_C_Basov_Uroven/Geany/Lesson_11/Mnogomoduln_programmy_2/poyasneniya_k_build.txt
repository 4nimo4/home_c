
Файл build используеися для того чтобы отдельно вручную
не вводить команды получения объектных файлов а из них уже
сборки в отдельный исполняемый файл и его запуск через терминал
и добавим еще пердупреждения (вывод сообщей об ошибках) для компилятора
//---------------------------------------------------------------
В Windows используется раширение для этого файла .bat:
//---------------------------------------------------------------
    build.bat

и в него помещаются команды:
    @echo off                           //— это команда, используемая в пакетных файлах Windows (.bat), 
                                        //чтобы скрыть отображение команд в окне командной строки при их выполнении.
                                        //Объяснение: echo off - отключает вывод команд на экран 
                                        //(но сами команды продолжают выполняться).
                                        //Символ @ - перед командой означает, что именно эта строка тоже не будет отображаться.
                                        //Зачем это нужно? Чтобы окно выглядело аккуратно и пользователь видел 
                                        //только нужные сообщения, а не каждую выполняемую строку кода.
                                        //Пример: 
                                        //@echo off
                                        //echo Привет, мир!
                                        //На экране появится только: Привет, мир!

    REM Компиляция с предупреждениями   //REM или :: - Так в исполняемом файле обозначаются коментарии 
                                        //они игнорируются компилятором
    gcc -Wall -c -o main.o main.c       //получение объектного файла main.o из main.c с предупреждениями -Wall
    gcc -Wall -c -o func.o func.c       //получение объектного файла func.o из func.c с предупреждениями -Wall

    REM Получение ассемблерного кода
    gcc -S -o main.s main.c             //получение ассемблерного файла из main.c с предупреждениями -Wall
    gcc -S -o func.s func.c             //получение ассемблерного файла из func.c с предупреждениями -Wall

    REM Связывание
    gcc -Wall -o prog.exe main.o func.o     //получение исполняемого файла prog из файлов main.o и main.c 
                                            //с предупреждениями -Wall

     
Далее в терминале вводим команду:
    build.bat                   //Скрипт собирает исполняемый файл
    prog.exe                    //Запуск исполняемого файла

//---------------------------------------------------------------
В Linux используется раширение для этого файла .sh:
//---------------------------------------------------------------
    build.sh        //Скрипт 

и в него помещаются команды:
    #!/bin/bash                 //— это так называемый шебанг (shebang). 
                                //Она указывает операционной системе, что этот скрипт 
                                //нужно запускать с помощью интерпретатора Bash, 
                                //который находится по пути /bin/bash
                                //Это обеспечивает правильное выполнение команд 
                                //внутри скрипта и позволяет запускать его как исполняемый файл.

    # Компиляция с предупреждениями // # - Так в исполняемом файле обозначаются коментарии они игнорируются компилятором
    gcc -Wall -c -o main.o main.c   //получение объектного файла main.o из main.c с предупреждениями -Wall
    gcc -Wall -c -o func.o func.c   //получение объектного файла func.o из func.c с предупреждениями -Wall

    # Получение ассемблерного кода  
    gcc -S -o main.s main.c         //получение ассемблерного файла из main.c с предупреждениями -Wall
    gcc -S -o func.s func.c         //получение ассемблерного файла из func.c с предупреждениями -Wall

    # Связывание                    
    gcc -Wall -o prog main.o func.o //получение исполняемого файла prog из файлов main.o и main.c с предупреждениями -Wall
 
Далее в терминале вводим команду:
    chmod +x build.sh          // Данная команда chmod +x делает Скрипт исполняемым
    ./build.sh                 //Скрипт собирает исполняемый файл
    ./prog                     //Запуск исполняемого файла
//---------------------------------------------------------------
