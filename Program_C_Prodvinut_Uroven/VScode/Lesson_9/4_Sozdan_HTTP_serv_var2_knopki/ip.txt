браузер все показывает! есть вопрос, почему для отображения в браузере 
нужно вводить именно 127.0.0.1:1500 а не например 192.168.0.0:1500 ?

Коротко: потому что ваш сервер сейчас слушает на этом компьютере, и:

127.0.0.1
— это loopback / localhost, гарантированно существующий адрес внутри машины.
192.168.0.0
— это сетевой адрес подсети, а не адрес конкретного хоста, обычно по нему вообще никто не “живет”.
Разберём по пунктам.

1. Что означает
INADDR_ANY
в коде
В сервере вы сделали:

server_addr.sin_addr.s_addr = htonl(INADDR_ANY);
INADDR_ANY
= «слушать на всех локальных интерфейсах»:

127.0.0.1 (loopback)
192.168.x.x (локальная сеть, если есть)
10.x.x.x, 172.16.x.x и т.п., если есть
и т.д.
То есть сам сервер не привязан конкретно к 127.0.0.1, он принимает соединения на всех IP, принадлежащих этой машине.

2. Почему работает
127.0.0.1:1500
127.0.0.1
(или
localhost
) — это специальный адрес:

всегда существует на любой машине;
всегда указывает на эту же машину;
никогда не выходит в сеть.
Вы запускаете сервер на своей Kubuntu → он слушает порт 1500 → вы из браузера на этой же Kubuntu подключаетесь к
127.0.0.1:1500
→ соединение идёт внутри системы и попадает в ваш сервер.

3. Почему не работает
192.168.0.0:1500
Сетевой диапазон
192.168.0.0/24
обычно выглядит так:

192.168.0.0
— адрес сети (служебный, не хост)
192.168.0.1
— обычно роутер
192.168.0.2
–
192.168.0.254
— конечные устройства
192.168.0.255
— широковещательный адрес
То есть:

192.168.0.0
— это не адрес конкретного компьютера, а идентификатор всей подсети.
Браузеру просто некуда к нему подключаться → соединение не устанавливается.
Если у вашей Kubuntu есть IP в этой сети, например
192.168.0.25
, тогда из другого устройства в сети можно обратиться к:

http://192.168.0.25:1500/
а с этой же машины — и к нему, и к
127.0.0.1:1500
.

4. Как узнать свой “настоящий” IP вместо 127.0.0.1
Чтобы из другого компьютера в сети открыть ваш сервер, нужно знать реальный адрес вашей Kubuntu:

ip a
Ищите строчку вида:

inet 192.168.0.25/24 ...
Тогда с другого ПК/телефона в той же Wi‑Fi сети можно вбить:

http://192.168.0.25:1500/
и увидеть ту же страницу (если нет блокировок firewall).

Если хотите, могу помочь:

объяснить, какой именно ваш IP сейчас, по выводу
ip a
;
настроить так, чтобы сервер был доступен с других устройств в сети.
